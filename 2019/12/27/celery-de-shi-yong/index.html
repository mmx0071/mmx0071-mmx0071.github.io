celery的使用
无标签
发布日期:   2019-12-27
更新日期:   2019-12-27
文章字数:   464
阅读时长:   1 分

celery是使用python语言制作出的分布式系统，主要是针对异步的任务进行处理，使用celery又被叫做生产者与消费者的设计模式，也就是说celery分为三个板块，分别为生产者，中间人，以及消费者，通俗的讲就是我们将需要异步处理的任务交给celery，这就形成了一个消费者，

from celery import Celery  #需要提前下载celery，并且使用celery的redis的版本必须超过3.xx的版本，redis的版本过低的话可以通过 pip uninstall redis 卸载，在通过 pip install redis 安装，安装后的版本即可满足需求

celery_app = Celery('meiduo')  # 创建一个celery对象

celery_app.config_from_object('celery_tasks.config') #导入celery的配置文件， 一般为broker（中间人）的地址，即celery服务器的地址

celery_app.autodiscover_tasks([任务名]) #任务以列表的形式添加

在使用celery的时候，可能会出现connection refused的错误，这个错误的出现有一定的偶然性，在我添加第二个任务交给celery执行的时候，就出现了这个错误，最终我添加第三个任务时，第二三都能成功，删除第三个任务后，错误消失，这之间第二个任务的代码并没有去改动。初步分析的结果是通过terminal开启celery，在关闭更新任务时，没有彻底的结束完，可以多使用几次ctrl + c来终止，也可以通过kill来杀死。

celery的进程数是跟自身的CPU有关的，双核即为8， 四核即为16，但这个可以通过一些设置来更改

celery worker -A proj --concurrency=进程数

也可以在eventlet 模块的作用下将进程更改为协程

# 安装eventlet模块
$ pip install eventlet

# 启用 Eventlet 池
$ celery -A celery_tasks.main worker -l info -P eventlet -c 1000
