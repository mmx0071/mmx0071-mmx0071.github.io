Django中的ORM
无标签
发布日期:   2019-12-27
更新日期:   2019-12-27
文章字数:   955
阅读时长:   3 分

现在比较普遍的程序设计模式一般为MVT，其中M指的便是Model，对于model我们一般是通过ORM框架来操作的，O是object，R是relation，M是mapping，即类对象数据库映射，通过类和类对象就能够操作它所对应的表格中的数据

在使用ORM的时候需要先配置数据库的连接信息

DATABASES = {
    'default': {
        'ENGINE': 'django.db.backends.mysql',
        'HOST': '127.0.0.1',    #库所在的地址，127.0.0.1为本机
        'PORT': 3306,           #mysql的默认端口为3306
        'USER': '数据库拥有者',
        'PASSWORD': '密码',
        'NAME': '库名',
    }
}

然后需要在models.py中定义模型类

之后要安装Pymysql驱动程序

pip install PyMySQL

在 Django 的工程同名子目录的 init.py 文件中添加如下语句

from pymysql import install_as_MySQLdb

install_as_MySQLdb()

Django提供自带的用户模型类，若是需要添加新的字段则可以用继承的方式，增加字段。

在定义完模型类后需要将数据迁移到数据库中

python manage.py makemigrations  #将需要迁移的数据生成文件

python manage.py migrate  #运行迁移文件生成对应的表格

每当我们对模型类进行修改之后都需要将上面两项命令重新执行一遍

在迁移有外键的表时需要注意迁移的顺序，必须先将外键所对应的表迁移后，才能进行迁移

数据库的表名：模型类如果未指明表名，Django 默认以 小写 app应用名_小写模型类名 为数据库表名。可通过 db_table 指明数据库表名

定义表属性时需要指定字段类型，通过字段类型的参数指定选项，语法如下：

属性=models.字段类型(选项)

通过ORM的映射可以对数据库进行增删改查等操作，这种操作一般都可以通过两种方法来实现，一种是通过类.objects来操作，另一种则是创建对象后，通过对象来操作，通过对象对数据进行处理后需要使用对象.save（）方法来对数据进行保存

数据库的操作上，查为最频繁的操作，在ORM中查询有这样几种：

1.get 查询单一结果，如果不存在会抛出 模型类.DoesNotExist 异常

2.all 查询多个结果。

3.count 查询结果数量，

4.filter过滤查询，filter（条件）可以过滤出多个结果

5.exclude也是过滤，符合条件的筛选掉

对于条件，大体可以分为：

exact：表示判等

contains：是否包含

startswith、endswith：以指定值开头或结尾

isnull：是否为null

in：是否包含在范围内

gte 大于等于 (greater then equal)

lte 小于等于 (less then equal)

year、month、day、week_day、hour、minute、second：日期也可以判断

F 对象 是对比两列时使用的方法

Q 对象是对数据进行逻辑操作时进行的方法

关联查询：

一对应的模型类对象.多对应的模型类名小写_set
多对应的模型类对象.外键名
多对应的模型类对象.关联类属性_id

关联过滤查询

关联模型类名小写__属性名__条件运算符=值 
一模型类关联属性名__一模型类属性名__条件运算符=值

两大特性

惰性执行：创建查询集不会访问数据库，直到调用数据时，才会访问数据库，调用数据的情况包括迭代、序列化、与if合用

缓存：使用同一个查询集，第一次使用时会发生数据库的查询，然后Django会把结果缓存下来，再次使用这个查询集时会使用缓存的数据，减少了数据库的查询次数
